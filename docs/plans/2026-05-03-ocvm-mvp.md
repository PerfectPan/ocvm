# OCVM MVP Implementation Plan

**Goal:** Build a publishable Rust CLI for OpenClaw version management.

**Architecture:** Use a native Rust binary with a small library core. CLI parsing lives in `src/cli.rs`; durable state and behavior live in separate modules for config, paths, project pinning, source providers, and version operations. This keeps the MVP simple while leaving room for snapshot, rollback, per-version config isolation, and checksum policies.

**Tech Stack:** Rust stable, `clap`, `serde`, `serde_json`, `reqwest` with `rustls`, `anyhow`, `assert_cmd`, `tempfile`.

---

## Deliverables

- Cargo package named `ocvm`
- Publishable binary target `ocvm`
- Full MVP command surface
- Source provider abstraction for npm and release manifests
- Isolated install layout under `~/.ocvm/versions/<version>`
- Project pin support via `.openclaw-version`
- Doctor diagnostics with actionable repair suggestions
- Integration tests and core behavior tests
- CI for fmt, clippy, and test

## Module Plan

```text
src/main.rs       process entrypoint
src/cli.rs        clap command declarations and dispatch
src/config.rs     config load/save and source kind
src/paths.rs      OCVM_HOME layout and path helpers
src/project.rs    .openclaw-version lookup/write/remove
src/source.rs     SourceProvider trait, npm provider, release provider
src/version.rs    install, uninstall, use, exec, list, doctor
```

## Test Plan

- `tests/core.rs`
  - install stages, verifies, and creates shim
  - failed install preserves existing version
  - project pin wins over global default
  - remote versions can be filtered by channel

- `tests/cli.rs`
  - help works without initializing state
  - default/current/list/use/uninstall lifecycle
  - project pin resolution through CLI
  - explicit `exec` does not change default
  - damaged default does not block explicit healthy version

## Verification Gates

```bash
cargo fmt --check
cargo clippy --all-targets -- -D warnings
cargo test
```

Network access is required once to download crates.
